# Claude Code Digest — 2026-05-24 00:38:06

## Version updates
Release 2.1.150 shipped on May 23. Anthropic lists only internal infrastructure improvements. You will notice no user-facing changes.

## What the docs reveal

### Autonomous Delegation Demands Explicit Trust
Anthropic is tackling developer fatigue. Constantly approving tool executions breaks flow, especially when agents handle repetitive background maintenance. The updated `allowedTools` configuration eliminates this friction. Developers can now whitelist specific tools to bypass manual prompts.

More significantly, Anthropic added wildcard support for Model Context Protocol (MCP) servers. By specifying `mcp__servername__*`, you grant blanket execution rights to all tools provided by that server. This design choice treats individual MCP servers as atomic units of trust. You can fully automate a local file-management server while keeping a production database server safely behind manual approval barriers.

This exact explicit trust model now governs subagents. Developers must explicitly include `Agent` in their custom agent's `allowedTools` array. Claude Code refuses to delegate tasks in `dontAsk` mode unless you declare this capability upfront. Anthropic implemented this restriction to prevent runaway recursive agent spawning. You must deliberately authorize your software to spin up autonomous subordinates.

### Subagent Sandboxing with Git Worktrees
Delegating complex tasks to subagents previously risked a polluted Git state. If an autonomous "code reviewer" or "test runner" hallucinated mid-refactor, it destroyed your working directory. You had to discard the broken state manually.

The new `isolation: worktree` configuration solves this problem. Subagents now execute inside temporary, isolated Git worktrees. Rather than rewriting your active `HEAD`, Claude Code branches the subagent strictly off your repository's default branch. You can override this behavior by setting `worktree.baseRef` to `"head"`, but the default isolates your uncommitted work out of the box.

Anthropic designed these worktrees to self-destruct. If the subagent fails or completes its task without modifying files, Claude Code automatically purges the directory. This isolation layer gives you the confidence to assign deeply exploratory tasks—like dependency upgrades or experimental refactoring—without fearing collateral damage.

### Persistent Sessions with Sheddable Context
Long-running interactive sessions routinely exhaust token limits. The agent accumulates dead context from solved problems, which degrades performance and inflates API costs. To address this, Anthropic exposed the `/clear` command directly within the Agent SDK.

Calling `/clear` immediately truncates the active conversation payload sent to the LLM. Crucially, Claude Code preserves the raw session history on disk. You wipe their active memory but systematically keep the audit trail. 

The documentation clarifies that `/clear` exclusively serves streaming input modes. One-shot `query()` calls already execute with a sterile context. By integrating this command into streaming workflows, Anthropic enables developers to build persistent background agents. These daemons can idle in your terminal, clear their context between standalone tasks, and hold onto a single stable session ID forever.
