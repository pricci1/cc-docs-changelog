# Claude Code Digest — 2026-03-17 20:16:21

## What the docs reveal

Anthropic updated the Claude Code documentation today. The changes shift how the CLI handles repository context and autonomous execution. Reading between the lines of the commits reveals a clear product direction: less manual context curation and stricter guardrails for agentic behavior.

**The death of manual AST parsing**
Anthropic deprecated the `--local-parse` flag. Claude Code now maps your project's Abstract Syntax Tree (AST) automatically before compiling its context prompt. 

Previously, developers toggled local parsing to save API tokens. Building the AST locally meant sending fewer, highly targeted snippets to the model. Anthropic clearly recognized this trade-off no longer makes sense. With persistent prompt caching and drastically reduced input costs, the bottleneck shifted from API limits to local CPU boundaries. Forcing the CLI to parse massive codebases locally caused command latency. Now, Claude Code sends a lightweight structural map, letting the cloud handle the heavy lifting.

For daily usage, this eliminates a persistent friction point. You no longer need to decide between speed and accuracy when asking Claude to refactor across multiple modules. The CLI pulls the right context automatically. Stop micromanaging your file selections.

**Strict sandboxing for autonomous test execution**
The documentation introduces a new `--sandbox-strict` flag for the `claude run` and `claude test` commands. This flag blocks outbound network requests and restricts file system writes to an isolated temporary directory.

User feedback undoubtedly drove this change. As developers increasingly rely on Claude Code to autonomously execute test loops and fix failing builds, the risk of destructive hallucinations rises. A runaway agent executing `npm install` on an unvetted, hallucinated package presents a severe supply chain vulnerability. Code rewriting itself is useful; code rewriting its own dependencies is dangerous.

Implementing this flag signals Anthropic’s ambitions for enterprise CI/CD pipelines. You can now let Claude Code debug complex state issues locally without fearing corrupted environments. If you integrate Claude Code into your automated workflows, make this sandbox flag your default. 

**Hierarchical context management**
New deployment guides detail a hierarchical inheritance model for `.claudecodeignore` files. Nested directories can now override root-level ignore rules. 

Large monorepos break global ignore paradigms. A single root-level ignore file works beautifully for a standalone Node.js service. It fails entirely when a repository contains a Next.js frontend, a Rust backend, and shared protobuf definitions—all requiring different context filters. Anthropic engineers likely saw large, complex projects dropping logic because the CLI choked on irrelevant build artifacts from secondary workspace packages.

This change enables granular context boundaries. You can ignore all `tests` globally, but explicitly include `frontend/tests` by placing a specific `.claudecodeignore` in that directory. Use this feature to silo your project domains. Clean context shapes better code.
