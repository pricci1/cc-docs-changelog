# Claude Code Digest — 2026-05-25 00:39:31

## What the docs reveal

### Preserving Context with Hot Authentication
The quickstart documentation introduces a mid-session `/login` command, allowing developers to re-authenticate or swap accounts without restarting the agent. 

Anthropic built this to solve a brutal friction point: lost context. Developers frequently juggle personal and enterprise Anthropic tiers, or operate under strict corporate token expiration policies. Previously, an expired JSON Web Token required tearing down the session. You lost your loaded files, the agent's working memory, and your prompt history. 

Hot authentication protects your context window. When a long-running refactoring task hits an authorization wall, you simply cycle your credentials and resume. This saves the time and token costs required to rebuild the agent's mental model. 

### Sandboxing Git Worktrees Securely
The sandbox now authorizes writes to the shared `.git` directory for linked worktrees, while keeping the `hooks/` and `config` folders strictly read-only. 

Git worktrees let developers check out multiple branches simultaneously without duplicating heavy dependencies like `node_modules` or `target`. Earlier iterations of Claude Code's sandbox choked on worktrees because they blindly blocked writes to the parent directory's `.git` folder, breaking standard commands like `git commit`. Anthropic fixed this by punching a precise hole in the sandbox. 

The engineering trade-off reveals Anthropic's maturity regarding threat models. They granted the exact access required for version control, but locked down `hooks/` and `config`. A hallucinating or hijacked agent cannot alter a repository's configuration or silently install a malicious `pre-commit` script. Developers gain seamless branch multitasking; attackers lose a trivial persistence vector.

### Inheriting Your Local Shell Environment
Claude Code now automatically sources your `.zshrc`, `.bashrc`, or `.profile` at session startup.

Until today, Claude Code executed commands in a sterile shell. If your workflow relied on custom bash functions, customized `PATH` variables, or proprietary internal toolchain aliases, the agent failed. It forced you to prompt with raw, fully-qualified absolute paths. Anthropic realized an effective AI assistant must adapt to the developer's local ecosystem rather than demanding a blank canvas. 

This fundamentally upgrades how you prompt the agent. You no longer need to explain your company's convoluted build scripts. You simply tell Claude Code to use your existing `make-prod` or `deploy-dev` aliases. Tapping into your dotfiles cuts down daily token overhead, accelerates task execution, and transforms the agent into a truly native pairing partner.
