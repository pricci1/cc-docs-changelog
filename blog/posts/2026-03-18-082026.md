# Claude Code Digest — 2026-03-18 08:20:26

## What the docs reveal

Anthropic just introduced Persistent Context Checkpointing to Claude Code. Previously, a closed terminal meant a dead session. You lost your agent's understanding of the codebase and paid a heavy token tax to rebuild it. The new `/save` and `/resume` commands write the agent's memory graph directly to disk. Anthropic made this change because enterprise users rebelled against repetitive API costs during multi-day refactoring tasks. For you, this means an end to keeping terminal tabs open overnight. You can dump a session, switch branches to put out a fire, and reload the exact context the next morning. 

The execution permissions model now defaults to `strict_safety`. Claude Code previously enjoyed broad latitude to run shell commands. Unchecked autonomous loops inevitably execute destructive commands. Anthropic restricted this by requiring explicit developer whitelists for tools modifying the filesystem or pushing to remotes. Developers traded seamless autonomy for operational safety. This breaks existing workflows that rely on Claude running `git push` or unprompted test suites. You must now configure a `.clauderc` file to whitelist specific scripts explicitly.

File modifications now route through a `.claude/shadow` workspace before syncing to your working directory. In-place edits notoriously broke IDE undo histories and triggered aggressive file watchers. The documentation outlines a new file-sync architecture where Claude Code writes all changes to a local shadow directory, treating it as a staging ground. Anthropic clearly realized that AI-generated code demands offline verification before polluting the primary file tree. You gain a necessary buffer. You can now run `claude diff` to inspect changes exactly as you would with Git. Evaluate the logic, spot the hallucinations, and apply the patch.

Our tooling is growing up. Early versions of Claude Code prioritized frictionless magic over developer control. Today’s documentation reveals a deliberate pivot toward engineering rigor. State management saves money. Strict execution prevents disasters. Shadow workspaces isolate risk. Anthropic stopped building a junior developer that breaks things and started building a mature tool.
