# Claude Code Digest — 2026-05-22 00:38:30

## Version updates

**2.1.147** brings architectural tweaks to session management and an update to prompt commands.

Pinned background sessions (`Ctrl+T`) now survive long periods of idleness and restart in place during Claude Code updates. Under memory pressure, the system sheds non-pinned sessions first. Anthropic clearly envisions developers running fleets of long-lived agents for asynchronous tasks. You can now trust that your pinned supervisor agent will survive system resource spikes.

The team also renamed `/simplify` to `/code-review`. The command targets correctness bugs at adjustable effort levels. This shift abandons abstract refactoring in favor of concrete, functional code auditing.

## What the docs reveal

### Enterprise Lockdown and Compliance
Administrators gain sweeping control over corporate deployments this week. New flags like `strictPluginOnlyCustomization` and `disableAgentView` force Claude Code to load configurations strictly from managed settings or restrict features globally. Furthermore, organizations can lock down Model Context Protocol (MCP) servers using exact string matching, wildcards, and centralized allowlists (`allowedMcpServers`). For cross-platform environments, Windows Subsystem for Linux (WSL) now inherits managed settings directly from the Windows registry.

*Why?* Enterprises want autonomous LLM tooling, but fear rogue MCP servers exfiltrating proprietary code or prompt injections executing arbitrary local commands. By forcing sandbox compliance, establishing an update floor (`minimumVersion`), and locking down configuration files, Anthropic makes Claude Code safe for strict corporate IT environments.

### Hooks Become Session Orchestrators
Hooks no longer rely on blunt exit codes to evaluate permissions. They now output structured JSON via standard output. This data payload allows hooks to inject initial user messages, modify watched directory paths, and dynamically shift permission modes during execution. For subagents, a new `continueOnBlock` flag lets a hook reject a specific action while keeping the teammate agent alive to try an alternative approach.

*Practical implication:* Developers can write lifecycle hooks that intelligently steer the agent. Instead of breaking the run when an agent attempts a risky command, your hook can deny the action, inject an explanatory JSON payload directly into the context, and prompt the agent to find a safer implementation.

### Legacy Models Dropped
Claude Code drops support for the Claude 3 series, Sonnet 4.5, and Opus 4.5. It now requires Sonnet 4.6, Opus 4.6, or the newly released Opus 4.7.

*Why?* Supporting older models bloats the codebase and fragments the user experience. Legacy models perform unpredictably on edge-case tool invocations and complex background orchestration. By enforcing a 4.6 baseline, Anthropic guarantees the reasoning capabilities necessary for high-fidelity tool execution.

### Sandboxing Matures
The `/sandbox` command now opens a dedicated dashboard with tabs for modes, overrides, and configurations. Anthropic introduced `@anthropic-ai/sandbox-runtime`, an isolation layer wrapping the entire process—including tools, hooks, and MCP servers. Administrators can globally block unsandboxed commands, restrict file read access paths, and limit network domains. 

*Practical implication:* If you run unattended automation runs using `--dangerously-skip-permissions`, you must pair it with strict containerization or the new process-wide runtime. Local boundary enforcement is no longer a suggestion; it represents the core design philosophy for agent security.

### Auto Mode Remains Anthropic-Exclusive
Auto mode replaces traditional user approval prompts with a specialized background classifier model to review actions. The documentation clarifies that this feature operates exclusively on the Anthropic API. Users routing traffic through Bedrock, Vertex, or Foundry cannot enable it.

*Why?* Securing the perimeter against scope escalation and prompt injection requires low-latency, trusted infrastructure. Anthropic deploys this classification layer internally; pushing the classifier to third-party clouds introduces provider constraints they cannot yet mitigate.

### Provider-Specific Error Routing
Claude Code now dynamically routes 5xx error statuses to the relevant cloud provider's status page (Anthropic, Bedrock, Vertex AI, or Foundry) rather than defaulting to `status.claude.com`. The documentation also separates 529 "Overloaded" temporary errors from hard account quota limits. This streamlines troubleshooting during regional cloud outages.
